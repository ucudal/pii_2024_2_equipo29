◆ AddPlayer()

async Task Library.DiscordBot.DiscordCommands.AddPlayer ( InteractionContext context)
inline

Comando que permite agregar jugadores a la sala de espera utilizando Lobby. Intenta iniciar una partida cada vez que es ejecutado.

Parámetros
contextEl contexto del usuario y el lugar donde fue utilizado el comando.

El jugador que ejecuta el comando se agrega a la sala de espera para participar en la partida. Tras agregar al jugador a la sala de espera, se intenta iniciar la partida si hay suficientes jugadores. Si el intento de iniciar la partida tiene éxito, se enviará una respuesta por DM de discord y en el canal donde se ejecutó el comando mencionando el canal de batalla y los nombre de los jugadores emparejados. El sistema empareja jugadores en salas de batalla privadas, para evitar el desbordamiento de mensajes en un canal público. Si no hay suficientes jugadores para iniciar la partida, el sistema continuará esperando más jugadores.

◆ Attack()

async Task Library.DiscordBot.DiscordCommands.Attack ( InteractionContext context,
[Option("MoveSlot", "Movimiento a utilizar.")] string moveSlot )
inline

Comando para atacar a un pokemon enemigo durante una batalla.

Parámetros
contextEl contexto del usuario y el lugar donde fue utilizado el comando.
moveSlotNúmero que indica el movimiento a utilizar. Se debe enviar el valor 1, 2, 3 o 4.
  • El valor debe ser un número entero entre 1 y 4, donde cada número corresponde a un movimiento específico.
  • Si el número no está dentro de este rango, se envía un mensaje de error.

Requisitos para la ejecución exitosa del comando:

  • El usuario debe estar en un canal de batalla. De lo contrario, se envía un mensaje de error indicando que el usuario debe estar en un canal de batalla.
  • El juego debe estar en curso. Si la batalla no está en progreso, se envía un mensaje de error.
  • Si el comando se ejecuta correctamente, se cambia el turno y se muestra un mensaje de confirmación.

Comando para cambiar el pokemon seleccionado durante la batalla.

Parámetros
contextEl contexto del usuario y el lugar donde fue utilizado el comando.
pokemonNameNombre del pokemon a cambiar:
  • Debe existir en la lista de pokemons elegidos por el jugador al inicio de la batallao. En caso contrario, no se encontrará el pokemon y se enviará un mensaje de error.

  • Si los parámetros son correctos y es el turno del jugador, el pokemon será cambiado exitosamente y se enviará una confirmación.

◆ ChoosePokemon()

async Task Library.DiscordBot.DiscordCommands.ChoosePokemon ( InteractionContext context,
[Option("Pokemon", "Pokemon a elegir.")] string pokemonName )
inline

Comando para elegir un pokemon en la sala de batalla.

Parámetros
contextEl contexto del usuario y el lugar donde fue utilizado el comando.
pokemonNameNombre del pokemon a elegir.
  • El nombre del pokemon debe estar disponible en el catálogo y no debe estar baneado (en el catálogo aparece si fue baneado). En caso de no estar disponible o estar baneado, se envía un mensaje diciendo que el pokemon no fue encontrado.
  • Si el nombre del Pokémon no es válido o no está disponible, se enviará un mensaje de error.

Un pokemon es baneado cuando se cumple la siguiente condición:

  • Cuando no tiene al menos cuatro movimientos. Esto se debe a que tres movimientos se toman como Move del tipo normal y uno para Move del tipo especial.

Requisitos para la ejecución exitosa del comando:

  • El usuario debe estar en un canal de batalla donde haya una sala activa. Si no se encuentra en un canal adecuado, se enviará un mensaje de error indicando que debe estar en un canal de batalla.
  • El juego no debe haber comenzado aún. Si la batalla ya ha iniciado, se enviará un mensaje de error.
  • Si el Pokémon elegido es válido y la batalla no ha comenzado, se confirma la elección del Pokémon y, si aplica, se muestra una imagen relacionada con el Pokémon elegido.
  • Se muestra el turno actual de la partida, si está disponible.

◆ NextTurn()

async Task Library.DiscordBot.DiscordCommands.NextTurn ( InteractionContext context)
inline

Comando para cambiar el turno de la partida.

Parámetros
contextEl contexto del usuario y el lugar donde fue utilizado el comando.

El juego cambiará de turno automáticamente si ocurre una de las siguientes acciones:

  • Se ejecuta el comando /Change de forma exitosa.
  • Se ejecuta el comando /Attack de forma exitosa.
  • Se ejecuta el comando /Item de forma exitosa.

Este comando fue creado a causa del siguiente problema, cuando un pokemon de un jugador se encuentra afectado por Sleep, se quedó sin pociones y a su vez es el único pokemon que tiene disponible el juego no cambiará el turno de forma automática, porque no se cumplen ningunas de las 3 condiciones presentadas en la lista de forma exitosa.

Requisitos para la ejecución exitosa del comando:

  • El usuario debe estar en un canal de batalla. De lo contrario, se envía un mensaje de error indicando que el usuario debe estar en un canal de batalla.
  • El juego debe estar en curso para que el turno pueda ser cambiado. Si la partida no está iniciada, no se podrá realizar la acción.
  • Si el comando se ejecuta correctamente, se cambia el turno y se muestra un mensaje de confirmación.

◆ StartGame()

async Task Library.DiscordBot.DiscordCommands.StartGame ( InteractionContext context)
inline

Comando para iniciar la partida en una sala de batalla.

Parámetros
contextEl contexto del usuario y el lugar donde fue utilizado el comando.

El juego iniciará automáticamente si:

  • Todos los jugadores tienen la cantidad de MaxPokemons establecida en Player. Por defecto, 6.

Este comando fue creado con la finalidad de permitir a los jugadores realizar batallas con menos pokemon. Por ejemplo, en el caso de que uno de los jugadores quiera jugar con un pokemon y otro jugador con tres pokemons podrán hacerlo gracias a este comando.

Requisitos para la ejecución exitosa del comando:

  • El usuario debe estar en un canal de batalla. De lo contrario, se envía un mensaje de error indicando que el usuario debe estar en un canal de batalla.
  • Los jugadores deben elegir al menos un pokemon cada uno. De lo contrario, se enviara un mensaje de error.
  • Si la sala de juego no está activa, se notifica que el usuario debe estar en un canal de batalla para poder iniciar el juego.

◆ UseItem()

async Task Library.DiscordBot.DiscordCommands.UseItem ( InteractionContext context,
[Option("ItemIndex", "Index del item a usar.")] string itemIndex,
[Option("PokemonName", "Nombre del pokemon que va a utilizar el item.")] string pokemonName )
inline

Comando para usar un item sobre un pokemon en una sala de batalla.

Parámetros
contextEl contexto del usuario y el lugar donde fue utilizado el comando.
itemIndexEl índice del item que se va a utilizar:
  • Debe ser un índice válido en la lista de items del jugador en turno.
  • El item no debe estar agotado.
  • Debe tomar valores de 1, 2 o 3.
  • En caso de no encontrar la poción o que se halla agotado, envía un mensaje del error.
pokemonNameEl nombre del pokemon en el cual se utilizará el item:
  • Debe ser uno de los pokemons seleccionados por el jugador al inicio de la batalla.
  • El pokemon debe estar vivo para usar las pociones de FullHeal y SuperPotion. En el caso de Revive debe estar muerto.
  • En caso de no encontrar el pokemon, envía un mensaje del error.

Requisitos para la ejecución exitosa del comando:

  • La sala de juego debe existir en el canal donde se ejecutó el comando.
  • El juego debe estar en curso, de lo contrario, se notifica que no ha empezado.
  • Solo el jugador en turno puede ejecutar el comando, si no es su turno, se le indica que espere.
  • Si los parámetros son correctos, el item se aplica al pokemon y se emvía un mensaje de confirmación.

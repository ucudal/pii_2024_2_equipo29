\doxysubsection{Descripción detallada}
Representa la máquina de estados que gestiona el estado actual de un pokemon.

Se utiliza el patrón de diseño {\ttfamily State}, la maquina de estados maneja el estado actual de cada {\ttfamily \doxylink{class_library_1_1_pokemon}{Pokemon}}.

Dentro de la máquina de estados se encuentra un atributo que contiene el estado actual {\ttfamily Current\+State} donde guarda una instancia de una clase que implemente la interfaz {\ttfamily \doxylink{interface_library_1_1_states_1_1_i_pokemon_state}{IPokemon\+State}}.

Dependiendo del estado que tenga el {\ttfamily \doxylink{class_library_1_1_pokemon}{Pokemon}} va actuar de forma distinta al llamar a cualquier método de \doxylink{class_library_1_1_states_1_1_state_machine}{State\+Machine}.

Además aplica el principio OCP de SOLID, ya que en caso de querer agregar un nuevo estado, se debería crear una nueva clase que implemente {\ttfamily \doxylink{interface_library_1_1_states_1_1_i_pokemon_state}{IPokemon\+State}}, agregar el nombre del estado en el {\ttfamily Enum\+State} y extender la clase {\ttfamily \doxylink{class_library_1_1_states_1_1_state_applier}{State\+Applier}} donde se agregan los estados a los pokemon. Esto permitiría agregar un efecto con un nuevo comportamiento sin modificar las clases pertinentes, extendiendo a las mismas. Esto permite que el código sea más mantenible y extensible a futuro.
